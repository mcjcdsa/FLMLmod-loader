# 发布指南（GitHub Pages 与静态托管）

本文说明如何将 `website/` 目录发布到线上，供他人访问。

## 方式一：GitHub Pages

1) 将本仓库推送至 GitHub  
2) 在 GitHub 打开仓库 → Settings → Pages  
3) Build and deployment:
   - Source: Deploy from a branch
   - Branch: 选择 `main` 分支，`/website` 子目录（或使用 `/root` 并把网站移到根目录）
4) 保存设置后，稍等 1-2 分钟，访问 `https://<你的用户名>.github.io/<仓库名>/`

注意：
- 如果使用自定义域名，按 GitHub 指引配置 DNS（CNAME）
- 页面资源使用相对路径，适合二级路径托管

## 方式二：静态对象存储（云厂商）

以「静态网站托管」方式上传 `website/` 中所有文件至对象存储（OSS/OBS/S3）并绑定域名。

关键点：
- 开启「静态网站托管」
- 首页文件配置为 `index.html`
- 禁止列目录（可选）

## 方式三：自建 Nginx / Apache

将 `website/` 内容部署到你的 Web 根目录：

```
server {
    listen 80;
    server_name your.domain;
    root /var/www/freedomland/website;
    index index.html;
}
```

## 验证清单

- 首页能正常打开（导航、样式、切换主题）
- 内页链接正常（快速开始、加载器、API、教程、打包与部署）
- 指向仓库内 Markdown 的链接可用（docs 与 mods）
